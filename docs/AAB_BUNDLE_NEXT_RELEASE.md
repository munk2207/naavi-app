## Foundational design principles (must be visible in every surface)

These are product-level rules — CLAUDE.md, the Naavi system prompt, and both mobile + voice-server client code must all respect them.

**P1 — Never act on the outside world without explicit positive approval.**
Applies to: SMS, WhatsApp, email, calendar events with attendees, voice messages, deletions — any action that touches someone else's phone/inbox/calendar.
Acceptable approvals: *"yes"*, *"approved"*, *"send it"*, *"confirm"*, *"go ahead"*.
Not acceptable: *"ok"*, *"sure"*, *"sounds good"*, silence, ambiguity — Naavi re-asks.
Unresolved inputs (*"my wife"* without a known contact, a date without a year, a place not verified) BLOCK the action until clarified. Never silent fallback. Never guess. Never default to user's own phone/email.
Internal actions (rule creation, memory writes, lookups, drafts, solo calendar events) flow naturally without approval.
