Naavi — Module List

Project: Naavi Personal AI Assistant Version: 2.0 Last Updated: March 20, 2026 Status: Live — Phase 8+
🤖 AI & Machine Learning
Module Version / Model Functionality Used In Status
Claude (Anthropic) claude-sonnet-4-6 Main AI brain for Naavi. Processes Robert's messages, generates spoken responses, decides which actions to take (DRAFT_MESSAGE, CREATE_EVENT, REMEMBER, SAVE_TO_DRIVE, etc.), extracts knowledge fragments, and extracts action items from conversation transcripts. naavi-client.ts, ingest-note, extract-actions Live
OpenAI Whisper whisper-1 Short-form voice transcription. Robert taps the red 🎙 button, speaks a memo, and Whisper converts the audio clip to text. Handles any language automatically. transcribe-memo Edge Function Live
OpenAI Embeddings text-embedding-3-small Converts knowledge fragments into 1536-dimensional vectors stored in pgvector. Enables semantic similarity search — Robert can ask about a topic and Naavi finds related memories even if exact words don't match. ingest-note, search-knowledge Edge Functions Live
AssemblyAI universal-2 SDK v4 Long-form conversation transcription with speaker diarization. Identifies who said what (Speaker A, Speaker B) so Robert can label Dr. Ahmed vs himself. Used for recording full medical or business conversations. upload-conversation, poll-conversation Edge Functions Live
🔵 Google Suite Integrations
Module API Functionality Used In Status
Google Calendar Calendar API v3 Syncs Robert's calendar events every 6 hours via a scheduled Edge Function. Powers the morning brief (today's events), travel time calculation, leave-time banner, and auto-start conversation recording when an appointment begins. sync-google-calendar, create-calendar-event, calendar.ts Live
Gmail Gmail API v1 Reads Robert's important unread emails for the morning brief. Sends emails on Robert's behalf when he asks Naavi to draft and send a message. Also caches sender email addresses for contact lookup. sync-gmail, send-email Edge Functions, gmail.ts Live
Google Drive Drive API v3 Saves notes and documents created by Naavi (SAVE_TO_DRIVE action). Also saves full conversation transcripts with action items. Robert can search his Drive from the Notes screen. References are stored in naavi_notes table. save-to-drive, search-google-drive, drive.ts Live
Google Contacts (People API) People API v1 Looks up contact email addresses by name when Robert asks to email someone. Used as the third tier in the contacts lookup chain: local contacts table → Gmail sender cache → People API. lookup-contact Edge Function, contacts.ts Live
Google Maps Distance Matrix API Calculates travel time from Robert's current GPS location to a calendar event destination. Powers the leave-time banner ("Time to leave — tap to navigate") and pre-fills directions in Google Maps. get-travel-time Edge Function, maps.ts Live
🟢 Backend & Database
Module Version Functionality Used In Status
Supabase Cloud (Free tier) Full backend infrastructure: PostgreSQL database, authentication (Google OAuth), Row Level Security, Edge Functions (serverless Deno runtime), and real-time subscriptions. All data is user-scoped via RLS policies. All Edge Functions, supabase.ts Live
pgvector Supabase extension PostgreSQL vector extension that stores 1536-dimensional embeddings in the knowledge_fragments table. Enables cosine similarity search so Naavi can find semantically related memories using the search_knowledge_fragments RPC. knowledge_fragments table, search-knowledge Live
📱 Mobile & Frontend
Module Version Functionality Used In Status
React Native / Expo Expo SDK 51 Cross-platform mobile framework. The Naavi app runs on iOS, Android, and Web from a single codebase. Expo provides the build toolchain, over-the-air updates, and access to native device APIs. Entire mobile app Live
Expo Router v3 File-based navigation for the app. Handles routing between the main screen (index.tsx) and the Notes screen (notes.tsx). Uses the same conventions as Next.js App Router. app/index.tsx, app/notes.tsx Live
expo-speech Expo SDK 51 Text-to-speech engine that speaks Naavi's responses aloud to Robert. Uses the device's native TTS engine. Fires after every Naavi response when not suppressed. useOrchestrator.ts Live
Web Speech API Browser native Real-time voice recognition via the browser's built-in speech engine. Powers the green VoiceButton — Robert holds and speaks, words are transcribed in real-time and sent to Naavi. useVoice.ts, VoiceButton.tsx Live
MediaRecorder API Browser native Records audio clips in the browser using the device microphone. Powers both the short Whisper memo button (🎙) and the long-form conversation recorder (🩺). Outputs audio/webm blobs sent to Edge Functions. useWhisperMemo.ts, useConversationRecorder.ts Live
react-i18next v13 Internationalization framework. Naavi supports English and French UI strings. Language switching is handled through i18n.ts and translation JSON files. lib/i18n.ts, app/index.tsx Live
🚀 Deployment & Infrastructure
Module Plan Functionality Used In Status
Vercel Free tier Hosts the Naavi web app. Auto-deploys on every git push to the mobile/ repository. Live at naavi-app.vercel.app. Serves the React Native Web build so Robert can access Naavi from any browser without installing an app. Web deployment Live
Supabase Edge Functions Deno runtime 13 serverless functions handling all server-side logic: AI calls, Google API calls, transcription, embeddings, calendar sync, and email sending. Deployed via Supabase CLI. API keys never touch the client. supabase/functions/ Live
GitHub munk2207/naavi Source control for the mobile app. Every push to main triggers a Vercel deployment automatically. Edge Function code is deployed directly via CLI and not tracked in this repository. CI/CD pipeline Live
