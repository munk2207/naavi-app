🤖 AI & Machine Learning
Module Version / Model Functionality Used In Status
Claude (Anthropic) claude-sonnet-4-6 Main AI brain for Naavi. Processes Robert's messages, generates spoken responses, and decides which actions to execute: DRAFT_MESSAGE, CREATE_EVENT, FETCH_TRAVEL_TIME, SCHEDULE_MEDICATION, REMEMBER, SAVE_TO_DRIVE, ADD_CONTACT, and more. Follows strict JSON output rules with 5 critical action rules in the system prompt. Also extracts knowledge fragments and action items from conversation transcripts. naavi-client.ts, ingest-note, extract-actions Live
OpenAI Whisper whisper-1 Short-form voice transcription. Robert taps the red 🎙 button, speaks a memo (minimum 1500ms guard to prevent double-fire), and Whisper converts the audio clip to text. Handles any language automatically. transcribe-memo Edge Function Live
OpenAI Embeddings text-embedding-3-small Converts knowledge fragments into 1536-dimensional vectors stored in pgvector. Enables semantic similarity search — Robert can ask about a topic and Naavi finds related memories even if exact words don't match. ingest-note, search-knowledge Edge Functions Live
AssemblyAI universal-2 SDK v4 Long-form conversation transcription with speaker diarization. Identifies who said what (Speaker A, Speaker B) so Robert can label Dr. Ahmed vs himself. Used for recording full medical or business conversations. upload-conversation, poll-conversation Edge Functions Live
🔵 Google Suite Integrations
Module API Functionality Used In Status
Google Calendar Calendar API v3 Syncs Robert's calendar events every 6 hours via a scheduled Edge Function. Powers the morning brief (today's events), travel time calculation, leave-time banner, and medication schedule events. Individual calendar events are created for each dose in a SCHEDULE_MEDICATION cycle (💊 emoji prefix). sync-google-calendar, create-calendar-event, calendar.ts Live
Gmail Gmail API v1 Reads Robert's important unread emails for the morning brief. Sends emails on Robert's behalf when he asks Naavi to draft and send a message. Also caches sender email addresses for contact lookup (Tier 2 of contact resolution). sync-gmail, send-email Edge Functions, gmail.ts Live
Google Drive Drive API v3 Saves notes and documents created by Naavi (SAVE_TO_DRIVE action). Also saves full conversation transcripts with action items. Robert can search his Drive from the Notes screen. References are stored in naavi_notes table. save-to-drive, search-google-drive, drive.ts Live
Google Contacts (People API) People API v1 Looks up contacts by name or phone number. Used in two paths: (1) Name lookup — Tier 3 of contact resolution chain after local contacts table and Gmail cache. (2) Phone lookup — when Robert asks about a phone number, the app tries 4 number formats (dashes, digits, +1 prefix, +1 spaced) via searchContacts, which searches across names, emails, and phone numbers. lookup-contact Edge Function, contacts.ts Live
Google Maps Distance Matrix API Calculates real driving time from Robert's current GPS location to any destination. Triggered by Claude via FETCH_TRAVEL_TIME action — no keywords required, Claude uses natural language understanding. Shows duration, distance, and leave-by time. For general queries shows "leave now → arrive by X". For upcoming events shows exact departure time. Opens in Google Maps via card button (user gesture required by browser security). get-travel-time Edge Function, maps.ts Live
🟢 Backend & Database
Module Version Functionality Used In Status
Supabase Cloud (Free tier) Full backend infrastructure: PostgreSQL database, authentication (Google OAuth), Row Level Security, Edge Functions (serverless Deno runtime). All data is user-scoped via RLS policies. All Edge Functions use verify_jwt = false with internal auth via userClient.auth.getUser() — resolves JWT key rotation compatibility issues. All Edge Functions, supabase.ts Live
pgvector Supabase extension PostgreSQL vector extension that stores 1536-dimensional embeddings in the knowledge_fragments table. Enables cosine similarity search so Naavi can find semantically related memories using the search_knowledge_fragments RPC. knowledge_fragments table, search-knowledge Live
📱 Mobile & Frontend
Module Version Functionality Used In Status
React Native / Expo Expo SDK 55 Cross-platform mobile framework. The Naavi app runs on iOS, Android, and Web from a single codebase. Expo provides the build toolchain, over-the-air updates, and access to native device APIs. Entire mobile app Live
Expo Router v5 File-based navigation for the app. Handles routing between the main screen (index.tsx) and the Notes screen (notes.tsx). Uses the same conventions as Next.js App Router. app/index.tsx, app/notes.tsx Live
expo-speech Expo SDK 55 Text-to-speech engine for native iOS/Android. On web, the browser's Web Speech Synthesis API is used instead with voice priority: Microsoft Aria Online (Natural) → Google UK English Female → Google US English → Samantha. Waits for onvoiceschanged to ensure voices are loaded before selecting. Rate: 0.88. useOrchestrator.ts Live
Web Speech API (Input) Browser native Real-time voice recognition via the browser's built-in speech engine. Runs in continuous mode — accumulates finalParts[] across multiple results and delivers the full transcript on onend. The green VoiceButton is clickable during the 'speaking' state (only blocked during 'thinking'). useVoice.ts, VoiceButton.tsx Live
MediaRecorder API Browser native Records audio clips in the browser using the device microphone. Powers both the short Whisper memo button (🎙) and the long-form conversation recorder (🩺). Outputs audio/webm blobs sent to Edge Functions. A 1500ms minimum guard prevents double-fire on web. useWhisperMemo.ts, useConversationRecorder.ts Live
useOrchestrator Custom hook The central conversation engine. Manages per-turn state (ConversationTurn[]) so each conversation exchange has its own action cards — no card bleed between turns. Handles all action execution including the new FETCH_TRAVEL_TIME and SCHEDULE_MEDICATION actions. Includes phone number detection (10-digit numbers extracted from messages) and person name detection with stopwords filter. hooks/useOrchestrator.ts Live
react-i18next v13 Internationalization framework. Naavi supports English and French UI strings. Language switching is handled through i18n.ts and translation JSON files. lib/i18n.ts, app/index.tsx Live
⚡ Action System
Action Type What It Does Triggered By Status
DRAFT_MESSAGE Creates a complete email draft (to, subject, body). One-tap send via Gmail send-email Edge Function. write / draft / compose / send / email Live
CREATE_EVENT Creates a Google Calendar event with title, description, start/end times, and attendees. schedule / book / meeting / appointment Live
FETCH_TRAVEL_TIME Fetches real driving time via Google Maps Distance Matrix. Shows destination, duration, distance, leave-by time, and Open in Maps button. Claude decides when to emit this — no keyword list required. Claude natural language understanding Live
SCHEDULE_MEDICATION Handles complex on/off medication cycles. Claude emits one action with name, dose times, on_days, off_days, start_date, duration_days. Orchestrator calculates all active dates and creates individual Google Calendar events (💊 emoji) for each dose. medication schedule with on/off cycle Live
ADD_CONTACT Saves a new contact to both the contacts table (name + email) and people table (name + email + phone + relationship). name + email or phone given Live
DRIVE_SEARCH Searches Google Drive for files matching a query. Returns file cards with tap-to-open links. ask about any file / document Live
SAVE_TO_DRIVE Creates a Google Doc with the given content and saves a reference to naavi_notes table. save / note / store / write down Live
REMEMBER Ingests text into semantic long-term memory via Claude classification + OpenAI embeddings + pgvector storage. remember / keep in mind / note that Live
SET_REMINDER Saves a reminder with title and datetime to the reminders table. No UI card yet — stored silently. set reminder / alert / notification Live
LOG_CONCERN / UPDATE_PROFILE Saves health/social concerns and preference updates to the topics table. No UI card yet. concerns / preferences / profile changes Live
